# DIFARYX Architecture

## High-Level System Architecture

DIFARYX is organized as a layered scientific workflow intelligence platform.

```text
User
 ↓
Scientific Workspace
 ↓
Workflow Intelligence Layer
 ↓
Scientific Analysis Layer
 ↓
Evidence Layer
 ↓
Knowledge Layer
 ↓
Persistence Layer
```

## Frontend Layer

Technology:

* React
* TypeScript
* Vite

Responsibilities:

* Scientific workspace
* Data visualization
* Evidence presentation
* Workflow execution
* Session management

## Backend Layer

Technology:

* Python
* FastAPI

Responsibilities:

* Spectral processing
* Scientific calculations
* Peak fitting
* Evidence generation
* Reference matching

## Scientific Analysis Layer

Current Techniques:

* XRD
* XPS
* FTIR
* Raman

Future Techniques:

* XRF
* DSC
* TGA
* SEM
* TEM
* SAXS
* WAXS
* PDF Analysis

## Evidence Layer

Every scientific conclusion must originate from evidence.

Evidence Types:

* Raw observations
* Processed observations
* Statistical metrics
* Candidate matches
* Validation indicators

## Workflow Intelligence Layer

Future agentic layer responsible for:

* Experimental planning
* Gap detection
* Validation guidance
* Scientific reasoning
* Recommendation generation

## Persistence Layer

Stores:

* Sessions
* Evidence
* Decisions
* Workflow history
* Scientific memory

## Design Principles

* Evidence First
* Reproducibility First
* Explainability First
* Human-in-the-Loop
* Scientific Transparency
